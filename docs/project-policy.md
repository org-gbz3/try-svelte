# プロジェクト方針

## 1. 目的

このリポジトリは、小さなフロントエンド機能の実装を通じて Svelte を学ぶためのものです。

## 2. スコープ

- 対象はフロントエンドのみ。
- まずは次の Svelte 基本概念を優先する。
- components
- props と events
- stores
- lifecycle と reactivity
- routing の基礎（必要な場合のみ）

## 3. 依存関係ポリシー

- 原則としてサードパーティライブラリを追加しない。
- ライブラリ追加は次のいずれかを満たす場合に限定する。
- 実行またはビルドに必須である。
- 同等機能を安全に自作する負荷が学習目的に対して大きすぎる。
- ランタイム依存は可能な限り少なく保つ。
- 可能ならブラウザ標準 API と Svelte の標準機能を優先する。

## 4. セキュリティポリシー

- 開発環境は Node.js LTS を使う。
- 必要がない限りグローバルインストールを避ける。
- このリポジトリで sudo npm install は使わない。
- 可能な限り lockfile を使い、再現可能なインストールを行う。
- 依存関係アップデートはマージ前に内容を確認する。

## 5. コーディングポリシー

- ファイルは小さく保ち、読みやすさを優先する。
- 過度な抽象化より、明示的で単純な状態管理を選ぶ。
- コメントは意図が伝わりにくい箇所にだけ付ける。
- 早すぎる最適化は避け、まず明快さを優先する。

## 6. Copilot 利用ポリシー

- Copilot の提案は学習しやすい最小実装を優先する。
- 差分は小さくし、既存スタイルを維持する。
- 非自明な変更では次を明記する。
- 何を変更したか。
- なぜその変更が必要か。
- どう検証するか。

## 7. ドキュメントポリシー

- 重要な意思決定は markdown で記録する。
- セットアップ手順と実行手順を最新に保つ。
- 前提条件が変わったら方針文書を更新する。
